# 下拉框配置方案（最终版）

## 一、项目概述

本方案旨在将系统中所有硬编码的下拉框替换为可配置的方式，通过数据库表 `dropdown_config` 统一管理下拉框选项，提高系统的可维护性和灵活性。

## 二、已实现的下拉框

| 分类名称 | 下拉框ID | 所在文件 | 类型 | 状态 |
|---------|---------|---------|------|------|
| chat_check_source | source | Chat_Check_Add.cshtml | value和text一致 | 已实现 |
| chat_check_evaluation | evaluation | Chat_Check_Add.cshtml | value和text一致 | 已实现 |
| chat_check_add_brife | brief | Chat_Check_Add.cshtml | value和text一致 | 已实现 |
| warn_seconds | warn_sec | Warn_Rst.cshtml | value和text不一致 | 已实现 |
| kill_hours | kill_hour | FengHao_Rst.cshtml | value和text不一致 | 已实现 |

## 五、前端代码修改方案

### 1. 引入公共JS文件

确保所有页面都引入了 `public.js` 文件，因为 `loadDropdownOptions` 函数定义在该文件中：

```html
<script src="~/Scripts/public.js"></script>
```

### 2. 修改下拉框HTML

将硬编码的下拉框选项替换为只保留默认选项：

```html
<select id="dropdown_id" class="txt">
    <option value="">请选择</option>
</select>
```

### 3. 添加JavaScript代码

在页面加载时，使用 `loadDropdownOptions` 函数加载下拉框选项：

```javascript
$(function() {
    // value和text一致的下拉框
    loadDropdownOptions("category_name", "dropdown_id");
    
    // value和text不一致的下拉框
    loadDropdownOptions("category_name", "dropdown_id", true);
});
```

2. **修改前端代码**：
   - 确保所有页面引入了 `public.js` 文件
   - 替换硬编码的下拉框选项
   - 添加JavaScript代码加载下拉框数据

   - 打开各个页面，确认下拉框能正常加载数据
   - 测试下拉框的选择功能
   - 测试数据提交功能
4. **后续维护**：
   - 使用下拉配置管理页面，方便后续对下拉框选项的维护和更新
   - 如需添加新的下拉框选项，只需在数据库中插入新记录，无需修改前端代码

## 七、注意事项

1. **数据备份**：在执行SQL插入语句前，建议备份 `dropdown_config` 表数据
2. **执行顺序**：先执行SQL插入语句，再修改前端代码，确保下拉框能正常加载数据
3. **默认选项**：保留 `<option value="">请选择</option>` 作为默认选项，确保用户体验
4. **排序**：SQL语句中已设置了 `sort_order` 字段，确保下拉框选项按预期顺序显示
5. **状态**：所有下拉框选项的状态都设置为 `1`（启用），确保它们能被正常加载
6. **特殊字符**：对于包含特殊字符的选项值，已在SQL语句中进行了适当处理
7. **兼容性**：确保修改后的代码与现有系统兼容，不影响其他功能

本方案将系统中所有硬编码的下拉框替换为可配置的方式，通过数据库表统一管理下拉框选项，提高了系统的可维护性和灵活性。实施后，系统管理员可以通过下拉配置管理页面方便地维护和更新下拉框选项，而无需修改代码。
