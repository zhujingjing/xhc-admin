# 小火柴任务管理系统 - 核心流程设计

## 1. 系统架构

### 1.1 整体架构
- **前端**：基于现有管理员后台的前端框架，使用ASP.NET MVC视图
- **后端**：ASP.NET MVC控制器，BLL业务逻辑层
- **数据层**：SQL Server数据库
- **集成**：与现有系统通过API接口集成

### 1.2 模块划分
- **任务管理模块**：任务的创建、分配、执行、审核
- **模板管理模块**：任务模板的创建、编辑、删除
- **分类管理模块**：任务分类的管理
- **时间段设置模块**：任务自动触发的时间段设置
- **统计分析模块**：任务完成情况和绩效统计
- **权限管理模块**：基于现有权限系统的任务权限管理

## 2. 核心流程

### 2.1 任务生成流程

#### 2.1.1 自动触发任务生成

**流程描述**：
1. 现有系统（如提现系统）产生新数据
2. 现有系统调用任务管理系统的API接口
3. 任务管理系统检查当前时间是否在设置的时间段内
4. 如果在时间段内，直接生成任务
5. 如果不在时间段内，将任务信息存储到临时表
6. 定时任务在下次工作时间开始时，处理临时表中的任务

**涉及界面**：
- **API接口**：供现有系统调用，无界面
- **时间段设置界面**：管理员使用，设置任务触发的时间段

#### 2.1.2 手动添加任务

**流程描述**：
1. 客服或管理员登录系统
2. 进入任务管理页面
3. 点击"添加任务"按钮
4. 填写任务信息（名称、描述、优先级、分配人等）
5. 提交任务
6. 系统生成任务并分配给指定客服

**涉及界面**：
- **任务管理界面**：客服和管理员使用，添加和管理任务

#### 2.1.3 定期任务自动生成

**流程描述**：
1. 定时任务运行
2. 检查所有激活的任务模板
3. 根据模板的执行周期，判断是否需要生成任务
4. 如果需要，生成任务并分配给指定客服
5. 记录任务生成日志

**涉及界面**：
- **任务模板管理界面**：管理员使用，设置定期任务模板
- **任务管理界面**：查看生成的定期任务

### 2.2 任务执行流程

**流程描述**：
1. 客服登录系统
2. 进入任务面板
3. 查看待处理任务列表
4. 选择一个任务，点击"开始处理"
5. 任务状态变为"处理中"
6. 客服完成任务后，点击"完成任务"
7. 任务状态变为"已完成"
8. 系统通知管理员进行审核

**涉及界面**：
- **任务面板**：客服使用，查看和处理待办任务
- **任务详情页面**：客服使用，查看任务详情和执行任务

### 2.3 任务审核流程

**流程描述**：
1. 管理员登录系统
2. 进入任务审核页面
3. 查看待审核任务列表
4. 选择一个任务，查看任务详情和完成情况
5. 点击"审核通过"或"审核拒绝"
6. 如果审核通过，任务状态变为"已审核"，系统自动计算得分
7. 如果审核拒绝，任务状态变为"待处理"，通知客服重新处理
8. 支持批量审核操作

**涉及界面**：
- **任务审核界面**：管理员使用，审核任务完成情况
- **任务详情页面**：管理员使用，查看任务详情

### 2.4 任务统计流程

**流程描述**：
1. 管理员或客服登录系统
2. 进入任务统计页面
3. 设置统计参数（时间段、人员、任务类型等）
4. 点击"查询"按钮
5. 系统显示统计结果，包括任务完成情况、得分、绩效等
6. 支持导出统计报表

**涉及界面**：
- **任务统计界面**：管理员和客服使用，查看任务统计和绩效计算结果

## 5. 集成与接口

### 5.1 与现有系统的集成

- **API接口**：提供任务触发接口，供现有系统调用
- **数据集成**：共享用户表，实现单点登录
- **业务集成**：与现有审核系统集成，实现任务自动触发

### 5.2 API接口设计

#### 5.2.1 任务触发接口

**URL**：`/api/task/trigger`
**方法**：POST
**参数**：
- `taskName`：任务名称
- `categoryID`：分类ID
- `description`：任务描述
- `priority`：优先级
- `relatedID`：关联的现有系统记录ID

**返回**：
- `taskID`：生成的任务ID
- `status`：操作状态

#### 5.2.2 任务查询接口

**URL**：`/api/task/query`
**方法**：GET
**参数**：
- `userID`：用户ID
- `status`：任务状态
- `startDate`：开始日期
- `endDate`：结束日期

**返回**：任务列表

## 6. 技术实现要点

### 6.1 任务自动触发

- 使用Web API接口接收现有系统的触发请求
- 实现时间段检查逻辑，判断是否立即生成任务
- 使用临时表存储非工作时间的任务请求
- 开发定时任务，在工作时间开始时处理临时表中的任务

### 6.2 定期任务生成

- 使用外部桌面程序执行定时任务
- 每天到点自动检查任务模板，根据执行周期生成任务
- 支持按天、按周、按月的定期任务
- 处理临时表中存储的非工作时间任务请求

### 6.3 任务评分算法

- 基于任务标准分和完成时间计算实际得分
- 完成时间越短，得分越高
- 超过截止时间完成的任务得分降低
- 审核不通过的任务得分降低

### 6.4 批量操作实现

- 在前端实现多选功能
- 在后端实现批量处理逻辑
- 支持批量审核、批量分配等操作

### 6.5 性能优化

- 使用索引加速数据库查询
- 实现任务分页加载
- 优化定时任务执行效率
- 使用缓存减少数据库访问

## 7. 系统安全

- **数据安全**：使用参数化查询防止SQL注入
- **权限控制**：严格的角色权限管理
- **输入验证**：对所有用户输入进行验证
- **日志记录**：记录所有任务操作日志
- **数据备份**：定期备份任务数据

## 8. 部署与维护

### 8.1 部署方案

- 与现有管理员后台部署在同一服务器
- 使用IIS作为Web服务器
- 部署外部桌面程序执行定时任务
- 配置桌面程序的执行时间和参数

### 8.2 维护计划

- 定期备份数据库
- 监控系统运行状态
- 优化数据库性能
- 及时更新系统功能
