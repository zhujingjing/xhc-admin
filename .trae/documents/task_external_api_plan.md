# 任务管理系统 - 外部API接口实现计划

## 1. 任务分析

通过对现有代码的分析，我们了解到任务管理系统已经具备以下核心功能：
- 任务分类管理
- 任务模板管理
- 时间段设置
- 任务管理
- 任务审核
- 任务操作日志
- 任务统计
- 任务调度
- 任务自动生成

目前系统已经有一个外部触发任务生成的接口 `TriggerTaskGeneration`，但它只接受一个参数 `templateId`。我们需要设计一个新的接口，接收三个参数：
1. 业务类型
2. 业务ID
3. 业务参数（JSON键值对）

## 2. 实现计划

### 2.1 任务分解

1. **[ ] 任务1：在TaskController中添加新的外部API接口**
   - 优先级：P0
   - 依赖：无
   - 描述：
     - 添加一个新的Action方法，接收业务类型、业务ID和业务参数
     - 实现参数验证和错误处理
     - 调用BLL层的方法生成任务
   - 成功标准：
     - 接口能够正确接收三个参数
     - 能够返回标准的JSON响应
   - 测试要求：
     - 测试接口调用，验证参数接收和响应格式

2. **[ ] 任务2：在BLL/Task.cs中添加外部任务生成方法**
   - 优先级：P0
   - 依赖：任务1
   - 描述：
     - 添加一个新的方法，接收业务类型、业务ID和业务参数
     - 根据业务类型和业务参数确定任务模板
     - 生成任务并保存到数据库
   - 成功标准：
     - 能够根据业务参数生成正确的任务
     - 任务能够正确保存到数据库
   - 测试要求：
     - 测试方法调用，验证任务生成和保存

3. **[ ] 任务3：设计客户端调用代码**
   - 优先级：P1
   - 依赖：任务1
   - 描述：
     - 设计C#客户端调用代码
     - 设计JavaScript客户端调用代码
     - 提供调用示例和参数说明
   - 成功标准：
     - 客户端代码能够正确调用API
     - 能够处理API响应
   - 测试要求：
     - 测试客户端代码调用API

### 2.2 技术实现细节

1. **API接口设计**：
   - 路径：`/Task/ExternalTaskGeneration`
   - 方法：POST
   - 参数：
     - `businessType`：业务类型（字符串）
     - `businessId`：业务ID（字符串）
     - `businessParams`：业务参数（JSON字符串）
   - 响应：标准JSON格式，包含状态码和消息

2. **BLL方法设计**：
   - 方法名：`GenerateExternalTask`
   - 参数：
     - `businessType`：业务类型
     - `businessId`：业务ID
     - `businessParams`：业务参数（Dictionary<string, string>）
   - 功能：
     - 根据业务类型确定任务模板
     - 处理业务参数，生成任务名称和描述
     - 调用现有方法生成任务

3. **客户端调用代码**：
   - C#：使用HttpClient调用API
   - JavaScript：使用fetch或axios调用API

## 3. 实施步骤

1. 首先在BLL/Task.cs中添加`GenerateExternalTask`方法
2. 然后在TaskController中添加`ExternalTaskGeneration`接口
3. 最后设计并测试客户端调用代码

## 4. 风险评估

1. **业务类型与任务模板映射**：需要确保业务类型能够正确映射到任务模板
2. **参数验证**：需要对输入参数进行严格验证，防止恶意输入
3. **错误处理**：需要提供清晰的错误信息，便于外部系统调试
4. **安全性**：考虑添加API密钥验证，防止未授权调用

## 5. 预期成果

1. 一个新的外部API接口，能够接收业务类型、业务ID和业务参数
2. 相应的BLL方法，能够根据参数生成任务
3. 客户端调用代码示例，便于外部系统集成

## 6. 测试计划

1. 测试API接口的参数接收和响应格式
2. 测试不同业务类型的任务生成
3. 测试错误情况下的响应
4. 测试客户端代码的调用

## 7. 后续优化

1. 添加API密钥验证，提高安全性
2. 添加任务生成记录，便于追踪
3. 提供更多的业务类型与任务模板映射选项
