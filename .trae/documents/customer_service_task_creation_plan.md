# 客服自助任务创建功能 - 详细设计方案

## 项目概述
为客服人员提供自助创建任务的功能，特别是用于上传微信沟通记录截图，并与用户记录关联。后续还将支持其他类型的自助任务创建，确保系统具有良好的扩展性和兼容性。

## 详细实施计划

### [ ] 任务1: 数据库结构设计与修改
- **优先级**: P0
- **依赖项**: None
- **描述**:
  1. **分析现有用户记录表**:
     - 确认用户表结构，特别是用户UID字段的定义和存储方式
     - 分析用户表与其他表的关联关系
  2. **任务表扩展**:
     - 添加 `TaskType` 字段 (int)，用于区分任务类型
     - 添加 `CustomerServiceID` 字段 (int)，记录创建任务的客服ID
     - 添加 `UserUID` 字段 (varchar(50))，关联用户记录
     - 添加 `AttachmentURL` 字段 (varchar(255))，存储截图路径
     - 添加 `AttachmentName` 字段 (varchar(100))，存储截图文件名
     - 添加 `ExtraData` 字段 (text)，存储额外的任务数据（JSON格式）
  3. **创建任务类型表**:
     - 表名: `TaskType`
     - 字段: `TaskTypeID` (int, 主键, 自增), `TaskTypeName` (varchar(50)), `Description` (varchar(200)), `IsActive` (bit, 默认1), `CreateTime` (datetime, 默认getdate())
  4. **创建用户任务关联表**:
     - 表名: `UserTask`
     - 字段: `UserTaskID` (int, 主键, 自增), `UserUID` (varchar(50)), `TaskID` (int, 外键), `CreateTime` (datetime, 默认getdate())
- **成功标准**:
  - 数据库表结构正确创建，字段类型和约束设置合理
  - 能够通过用户UID查询关联的任务
  - 支持后续添加其他类型的自助任务
- **测试要求**:
  - `programmatic` TR-1.1: 数据库表结构正确创建，无语法错误
  - `programmatic` TR-1.2: 能够通过用户UID查询关联的任务
  - `programmatic` TR-1.3: 任务类型表能够正常添加和查询数据
- **备注**:
  - 新字段应设置默认值，确保现有任务记录不受影响
  - 为 `UserUID` 和 `TaskID` 字段创建索引，提高查询性能

### [ ] 任务2: 后端API接口开发
- **优先级**: P0
- **依赖项**: 任务1
- **描述**:
  1. **创建客服任务API**:
     - 接口路径: `POST /Task/CreateCustomerServiceTask`
     - 参数: `UserUID` (必填), `TaskTypeID` (必填), `TaskName` (必填), `Description` (可选), `Attachment` (文件, 可选)
     - 功能: 验证用户UID存在性，处理文件上传，创建任务记录，建立用户任务关联
  2. **获取任务类型列表API**:
     - 接口路径: `GET /Task/GetTaskTypes`
     - 参数: 无
     - 功能: 返回所有激活的任务类型
  3. **更新任务状态API**:
     - 接口路径: `PUT /Task/UpdateTaskStatus`
     - 参数: `TaskID` (必填), `Status` (必填), `OperatorName` (必填)
     - 功能: 更新任务状态，记录操作人
  4. **获取用户任务列表API**:
     - 接口路径: `GET /Task/GetUserTasks`
     - 参数: `UserUID` (必填)
     - 功能: 返回指定用户的所有任务
  5. **文件上传处理**:
     - 存储路径: `~/Uploads/TaskAttachments/`
     - 文件名格式: `{TaskID}_{时间戳}_{原文件名}`
     - 支持的文件类型: jpg, jpeg, png, gif
     - 文件大小限制: 5MB
- **成功标准**:
  - API接口能够正常接收和处理客服创建任务的请求
  - 能够正确关联用户记录
  - 支持文件上传功能
  - 接口返回标准的JSON格式数据
- **测试要求**:
  - `programmatic` TR-2.1: API接口返回正确的状态码和数据
  - `programmatic` TR-2.2: 能够正确关联用户记录
  - `programmatic` TR-2.3: 文件上传功能正常，文件路径正确存储
- **备注**:
  - 接口设计应考虑安全性，验证客服权限
  - 实现请求参数验证，防止恶意输入
  - 添加异常处理，返回友好的错误信息

## 技术实现细节

### 数据库设计
1. **任务表扩展SQL**:
   ```sql
   ALTER TABLE Task ADD 
       TaskType INT NULL,
       CustomerServiceID INT NULL,
       UserUID VARCHAR(50) NULL,
       AttachmentURL VARCHAR(255) NULL,
       AttachmentName VARCHAR(100) NULL,
       ExtraData TEXT NULL;
   
   -- 创建索引
   CREATE INDEX IX_Task_UserUID ON Task(UserUID);
   CREATE INDEX IX_Task_TaskType ON Task(TaskType);
   ```

2. **任务类型表SQL**:
   ```sql
   CREATE TABLE TaskType (
       TaskTypeID INT IDENTITY(1,1) PRIMARY KEY,
       TaskTypeName VARCHAR(50) NOT NULL,
       Description VARCHAR(200) NULL,
       IsActive BIT NOT NULL DEFAULT 1,
       CreateTime DATETIME NOT NULL DEFAULT GETDATE()
   );
   
   -- 插入默认任务类型
   INSERT INTO TaskType (TaskTypeName, Description) VALUES
   ('微信沟通记录', '客服上传微信沟通记录截图'),
   ('电话沟通记录', '客服记录电话沟通内容'),
   ('邮件沟通记录', '客服记录邮件沟通内容');
   ```

3. **用户任务关联表SQL**:
   ```sql
   CREATE TABLE UserTask (
       UserTaskID INT IDENTITY(1,1) PRIMARY KEY,
       UserUID VARCHAR(50) NOT NULL,
       TaskID INT NOT NULL,
       CreateTime DATETIME NOT NULL DEFAULT GETDATE(),
       FOREIGN KEY (TaskID) REFERENCES Task(TaskID)
   );
   
   -- 创建索引
   CREATE INDEX IX_UserTask_UserUID ON UserTask(UserUID);
   CREATE INDEX IX_UserTask_TaskID ON UserTask(TaskID);
   ```
